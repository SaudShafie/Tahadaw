Welcome. The name Tahadaw comes from the hadith “Tahadaw tahabbu” — give gifts, and you will love one another. That feeling is what we set out to protect. What makes it different

We don’t sell you things. We help you think.

Most “gift finders” are just shopping. Tahadaw starts with the idea.

First — a meaningful idea

The AI suggests thoughtful gift ideas made for the person and the occasion.

Then — the real product

Only once you love an idea do we go find the real, buyable gift.

This is the heart of Tahadaw. We deliberately separate the idea from the product. The AI reasons about what would actually be meaningful first — and we only go shopping once you’ve fallen for an idea. That one decision is why suggestions feel personal instead of like an ad.

For the technical reviewers: Tahadaw is a clean, layered backend. Requests flow through a security layer with encrypted passwords and role-based access, into controllers, into the business logic that runs the gift journey, down to a MySQL database. All the outside services — the AI, the shopping search, payment, WhatsApp, and email — plug in around it. The stack is Java 17 and Spring Boot 4.1, with QR generation and PDF support built in.

Tahadaw is designed for the cloud. It’s already packaged as a Docker image — the app plus everything it needs to run. The plan on AWS is straightforward: store the image in ECR, run it on ECS/Fargate where it can scale, with a managed MySQL database on RDS. Because gift cards are stored in the database rather than on disk, we can run many copies at once. Secrets stay outside the code, and we bundled the Arabic fonts so every card renders perfectly on a cloud server.
